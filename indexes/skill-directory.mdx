## 工作流 / 方法论

### obra/superpowers

- `brainstorming` — 创意、功能、组件或行为改动前先澄清意图、需求和设计方向。
- `dispatching-parallel-agents` — 面对两个以上彼此独立、无顺序依赖的任务时，拆给并行代理执行。
- `executing-plans` — 已有书面实现计划时，按检查点执行计划并接受审查。
- `finishing-a-development-branch` — 实现和测试完成后，决定合并、PR、清理等收尾路径。
- `receiving-code-review` — 收到代码评审意见后，先验证和分析，再决定如何处理。
- `requesting-code-review` — 完成任务或重大功能后，请求评审以确认满足需求。
- `subagent-driven-development` — 当前会话里按独立任务使用子代理推进实现。
- `systematic-debugging` — 遇到 bug、测试失败或异常行为时，先系统排查再修复。
- `test-driven-development` — 实现功能或修 bug 前，先写失败测试，再实现。
- `using-git-worktrees` — 需要隔离工作区时，创建和管理 git worktree。
- `using-superpowers` — 会话开始时查找并调用适用技能的总入口规则。
- `verification-before-completion` — 声称完成、修复或通过前，先运行验证命令并保留证据。
- `writing-plans` — 多步骤任务动手前，先把方案写成可执行计划。
- `writing-skills` — 创建、修改或验证技能时使用。
